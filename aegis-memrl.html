The Core Mechanism — Two-Phase Retrieval Replaces passive semantic search in Innovation #2
Phase A
Similarity
Recall
Retrieve the top-K₁=20 episodes by cosine similarity. This is identical to the existing Qdrant search in Innovation #2 — but casts a wider net (20 candidates instead of 4).
cosine(query_vector, z_i) → top K₁ = 20 candidates → semantic filter only
→
Phase B
Value-Aware
Selection
From the 20 Phase A candidates, select the top-K₂=4 by Q-value. High-utility experiences are promoted. Similar-but-low-quality noise is suppressed. This is what passive RAG cannot do.
score_i = 0.4·sim_i + 0.6·Q_i → top K₂ = 4 selected → quality-filtered context

The Intent-Experience-Utility Triplet
zi
Intent Embedding
nomic-embed-text vector of the original user query. Used in Phase A cosine similarity search. Already computed by Innovation #2.
embed("debug Python KeyError") → [0.023, -0.181, 0.447, ...] → 768-dimensional vector
ei
Experience
The compressed episode summary written by MemoryWriter after each turn. Contains what happened, the approach taken, and the outcome. Already stored in Qdrant.
"Debugged KeyError in data pipeline. Root cause: off-by-one in dict comprehension. Fixed with .get(). Outcome: resolved in 2 turns."
Qi
Learned Utility
The Q-value. Starts at 0.50 (neutral). Updated by implicit feedback via EMA. The only new field added to the existing Qdrant schema.
Q_new = 0.30 × reward + 0.70 × Q_old Initial: 0.50 After approved: 0.62 After 5 uses: 0.81
4 Wiring Changes to Existing Files
context_engine.py
Replace get_recent_episodes() with MemRLContextRetriever.get_episodes(). Two-Phase Retrieval activates automatically for every inference. The calling code in ContextEngine.build() doesn't change — only the retrieval method.
context_engine.py
MemoryWriter
After write_turn() stores an episode, call MemRLFeedbackCollector.on_episode_written(episode_id, outcome). The outcome field already exists — this just routes it to the Q-value updater.
puppeteer.py
AuditorPuppet
After the Auditor runs, call MemRLFeedbackCollector.on_auditor_complete(episode_id, auditor_output). The Auditor's JSON (approved, confidence, issues) provides the strongest quality signal without any human input.
agent_harness.py
HardenedPuppeteer
At the start of each new message, call MemRLFeedbackCollector.on_user_message(message). A user's follow-up = the previous response was good enough to continue (+0.70). A correction keyword = it wasn't (+0.15).
nightly.sh
Add one line after the context engine consolidation step: python3 ~/.aegis/memrl/memrl_engine.py nightly. Initialises Q-values for new episodes, marks pruning candidates, sends silent Telegram health report.
